# Notes-to-self — README

Working habits, retrospectives, and the "documenting behaviours" Nicolas mentioned — notes to himself about how he wants to work, what has worked, what to repeat or avoid.

What belongs here:
- working-method notes (what workflow worked on which kind of task)
- retrospectives (what went well / badly on a piece of work, lessons)
- behavioural reminders (habits to keep or drop)
- useful info worth keeping that isn't engineering knowledge or project record

This is the personal-process layer — it makes the LLM's advice more tailored over time without being either engineering knowledge or a project log.
